SPA 安全架構流程圖

PKCE 登入循序圖

sequenceDiagram
    autonumber
    participant User as 使用者
    participant SPA as 前端 (SPA)
    participant Auth as 授權伺服器
    participant API as 後端 API

    User->>SPA: 點擊登入
    SPA->>SPA: 產生 code_verifier 與 code_challenge
    SPA->>Auth: 導向登入 (攜帶 code_challenge)
    Auth-->>SPA: 回傳 Authorization Code
    SPA->>Auth: 發送 Code + code_verifier 請求換發 Token
    Auth->>Auth: 驗證 SHA256(verifier) == challenge
    Auth-->>SPA: 驗證成功，回傳 Access Token
    SPA->>API: 攜帶 Access Token 呼叫 API

CSP 防護流程圖

flowchart TD
    A[駭客嘗試進行 XSS 攻擊，注入惡意腳本] --> B{伺服器是否設定嚴格的 CSP?}
    B -- 否 --> C[瀏覽器執行惡意腳本]
    C --> D[讀取記憶體或 Storage]
    D --> E((Token 遭竊))
    
    B -- 是 --> F[瀏覽器依據白名單阻擋未授權腳本]
    F --> G((攻擊失敗，Token 安全))
