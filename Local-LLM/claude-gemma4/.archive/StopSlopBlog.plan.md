# 實作計畫：StopSlopBlog - blog.md 台灣用語與去 AI 味優化

本計畫旨在對 `blog.md` 進行台灣正體中文用語校正，並移除文章內可預測的 AI 寫作模式（AI slop），打破公式化結構與冗長清喉嚨詞彙，提升文章可讀性。

## 實作步驟

- [x] **步驟 1：執行用語掃描器對 `blog.md` 進行用語檢查** <!-- id: 1 -->
  - **說明**：使用 `stop-slop-zh-tw` 的 `zh-tw-terms.mjs` 掃描 `blog.md`，獲取命中中國用語（auto 與 flag 類）的完整清單，這能提供客觀的用語修改依據，避免直接修改遺漏。

- [x] **步驟 2：套用自動校正（--fix）與整理 flag 項目** <!-- id: 2 -->
  - **說明**：執行 `--fix` 自動修正 `auto` 類的安全用語，並將所有 `flag` 類的同形詞整理出來，以便在後續人工改寫時逐一確認。

- [x] **步驟 3：人工審查、改寫與去 AI 味** <!-- id: 3 -->
  - **說明**：讀取 `blog.md` 內容，針對 flag 用語依語境判斷替換。同時，清除「值得注意的是」、「不僅僅是...更是...」、「首先、其次、最後」等 AI 公式化贅詞，提升文章流暢度，符合台灣讀者閱讀習慣。

- [x] **步驟 4：進行 Stop-Slop 評分與微調** <!-- id: 4 -->
  - **說明**：依據 `stop-slop-zh-tw` 的六大維度（直接、節奏、信任、真實、密度、台灣味）對改寫後的文章進行 1-10 分的評分，若總分低於 42/60 則繼續修改。
  - **評分結果**：總分 53/60（直接 9/10, 節奏 8/10, 信任 9/10, 真實 9/10, 密度 8/10, 台灣味 10/10），符合標準。

- [x] **步驟 5：確認功能正確並更新 `@tree.md` 專案結構** <!-- id: 5 -->
  - **說明**：確認修改後的 `blog.md` 格式與排版正確（此為 markdown 檔案無須 build，但需要確認語意與 markdown 渲染正常），並同步更新 `@tree.md` 將當前的 `StopSlopBlog.plan.md` 加入結構中。

- [x] **步驟 6：封存計畫檔案** <!-- id: 6 -->
  - **說明**：完成所有步驟後，將 `StopSlopBlog.plan.md` 移動到 `.archive` 資料夾，並再次更新 `@tree.md`，保持根目錄乾淨。
